# 📦 Модуль обліку замовлень (ERP System)

Мікросервіс для бізнес-системи, що реалізує бекенд-функціонал обліку клієнтів, товарів та замовлень. Проєкт створено як тестове завдання з акцентом на надійність, чистоту коду та "продуктовий" підхід до розробки ERP-напрямку.

## 🛠 Технологічний стек
* **Бекенд:** Python 3.11, Django 5.0, Django REST Framework (DRF)
* **База даних:** PostgreSQL 15 (для забезпечення ACID-транзакцій)
* **Інфраструктура:** Docker, Docker Compose
* **Тестування:** Django `APITestCase`

## 💡 Архітектурні рішення та Бізнес-логіка
1. **Вибір фреймворку:** Обрано Django + DRF завдяки наявності потужного механізму серіалізаторів, які ідеально підходять для строгої типізації та валідації вхідних даних у фінансових/ERP системах.
2. **Точність розрахунків:** Для зберігання цін використано `DecimalField`, що виключає помилки округлення, притаманні `Float`.
3. **Оптимізація запитів:** У списку замовлень клієнта застосовано `prefetch_related('products', 'client')`, що запобігає проблемі N+1 запитів до бази даних.
4. **Виконання обов'язкових бізнес-правил:**
   * *Замовлення без клієнта неможливе:* Гарантується на рівні зв'язків БД (`ForeignKey`) та обов'язковим полем у DRF-серіалізаторі.
   * *Мінімум один товар у замовленні:* Контролюється через кастомний валідатор `validate_product_ids`.
   * *Автоматичний підрахунок суми:* Реалізовано на рівні бекенду. Сервер не приймає підсумкову суму з фронтенду, а самостійно розраховує її на основі актуальних цін у БД під час створення транзакції.

├── requirements.txt     # Залежності проєкту

🚀 Інструкція із запуску (через Docker)
Проєкт повністю докерізований, тому для запуску не потрібно локально встановлювати Python чи PostgreSQL.

1. Збірка та запуск контейнерів:
Виконайте в кореневій папці проєкту (де знаходиться docker-compose.yml):

Bash
docker compose up -d --build
2. Застосування міграцій до бази даних:

Bash
docker compose exec web python manage.py makemigrations orders
docker compose exec web python manage.py migrate
3. Створення адміністратора (опціонально):
Для доступу до вбудованої панелі керування (http://localhost:8000/admin/) створіть суперкористувача:

Bash
docker compose exec web python manage.py createsuperuser
🧪 Запуск тестів
У проєкті реалізовано тести для перевірки бізнес-правил (автоматичний підрахунок суми, валідація на порожній масив товарів). Для їх запуску виконайте:

Bash
docker compose exec web python manage.py test
📡 Формат API та приклади запитів
API доступне за базовою адресою: http://localhost:8000/api/

1. Створення клієнта
POST /clients/

JSON
{
  "name": "ТОВ Рога і Копита",
  "email": "contact@company.com"
}
2. Створення товару
POST /products/

JSON
{
  "name": "Серверна стійка",
  "price": "15500.00"
}
3. Створення замовлення
POST /orders/

JSON
{
  "client": 1,
  "product_ids": [1, 2, 3]
}
Зверніть увагу: Поле total_amount не передається. Воно розраховується та повертається сервером у відповіді автоматично.

4. Отримання замовлень клієнта
GET /clients/{client_id}/orders/
Повертає масив усіх замовлень конкретного клієнта з вкладеним описом кожного товару (із застосуванням оптимізації запитів).
